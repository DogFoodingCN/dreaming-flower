This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

Dreaming Flower is a Next.js App Router project for `dogfooding.cn`. The product positioning is personal space, IP building, product experience, and blog output.

Current MVP scope includes the galaxy-style homepage and a lightweight static MDX blog module. The homepage should act as a galaxy-style entrance for future modules; the blog provides writing output without CMS/database/account features. See committed constraints in `docs/constraints/` before expanding scope:

- `docs/constraints/project.md` — project identity, domain, file policy
- `docs/constraints/mvp.md` — MVP in-scope and out-of-scope boundaries
- `docs/constraints/visual.md` — galaxy visual direction, responsiveness, theme, motion



## Architecture

- `src/app/layout.tsx` defines the root App Router layout and metadata.
- `src/app/page.tsx` is the current MVP homepage.
- `src/app/blog/page.tsx` and `src/app/blog/[slug]/page.tsx` define the static MDX blog list and detail pages.
- `src/app/globals.css` contains Tailwind import, CSS variables, global theme colors, and homepage/blog visual styles.
- `content/blog/` contains local MDX blog entries.
- `public/` contains static assets generated by the initial Next.js scaffold.
- `docs/constraints/` contains commit-safe product and implementation constraints.

The app is intentionally small. Keep homepage-specific work in the home components and global styles until repeated structure justifies extraction; keep blog content as local MDX files under `content/blog/`.

## Project Constraints

- MVP includes the homepage and static MDX blog list/detail pages.
- Do not implement product detail pages, authentication, admin, CMS, database persistence, comments, subscriptions, user accounts, dynamic blog authoring, or full module routes unless the MVP constraints are explicitly changed.
- Future modules may appear on the homepage as non-final entry points, labels, descriptions, or coming-soon interactions.
- The homepage must remain responsive for mobile and desktop.
- The site must support day and night modes.
- Visual direction should stay galaxy-inspired: stars, orbits, nodes, constellations, and entrance-like spatial composition.
- Motion should be calm and should not reduce readability or basic accessibility.

## Model Fallback

- When a requested model channel is unavailable, fall back to the session default model instead of blocking. The current default model is `gpt-5.5`.

## File Policy

AI-generated planning, ideation, and draft files must stay inside this repository under `.ai/`, separated by purpose:

- `.ai/plans/`
- `.ai/ideas/`
- `.ai/drafts/`

These `.ai/` subdirectories are local-only and ignored by Git. Commit project constraints under `docs/constraints/` instead.
